SEBTL - Build Action History

Search

·         Audit Type:  Displays the type of audit.

·         Business Function Name:  Select the business activity to audit.

·         Page Name:  Enter the name (or part of the name) of the page to audit.

Assign Transaction Logs

·         Page Name:  Displays the name of the page located by the search. Select an unmarked checkbox to assign a log (i.e., record any page actions) to the corresponding page. Clear a marked checkbox to remove the log for that page.

SEVTL - View Action History

Search

·         Page Name:  Enter the name (or part of a name) of the needed page. If a page name is entered, the User ID and Case # must be provided also.

·         Case #:  Enter the case number of the relevant case.

·         Magnifying Lens:    Click to open the Search Criteria pop-up window to search for the needed case.

·         User ID:  Enter the worker’s unique ID .

·         Magnifying Lens:    Click to open the Search Criteria pop-up window to search for the worker needed client.

·         Batch ID:  Enter the identification of the needed batch process.

·         Case Action:  Select Add to locate a new record, Change to locate an edited record, or View to locate a viewed record.

·         Transaction Start Date:  Enter when the log should begin.

·         Time:  Select the hour and minutes the log should begin.

·         Transaction End Date:  Enter when the log should stop.

·         Time:  Select the hour and minutes the log should stop.

Action History Results

·         Time Stamp:  Displays when the action occurred.

·         User ID/Batch ID:  Displays the ID associated with the action.

·         Page Name:  Displays the name of the page where the action occurred.

·         Case Action:  Displays the type of action that occurred.

·         Case Details:  Displays either the case number, application number, person ID, participant/provider ID, or the fair hearing number.
