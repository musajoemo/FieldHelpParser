DCMEE - Medicare Claim – Details

Individual Information

·         Name:Select the individual’s name if this is a new record. Otherwise, displays the individual’s name.

·         Person ID:Displays the unique ID number assigned to an individual. This field populates after the Name field is selected.

·         Updated Date:Displays the date that the record was updated.

Medicare Claim Dates

·         Circumstances Start/Change Date:(CS/CD) Enter the start or change date for a new circumstance. Displays the first of the month, three months prior to the Application File Date for Intake case action. This is the date the individual in focus began (and stopped if applicable) receiving Medicare.

·         End Date: This field is disabled and will display the Medicare end date if one is entered on this page.

·         Reported On: Enter the date the client reported a change. Displays the Application File Date for Intake case action.

·         Date Client Became Aware:Enter the date the client became aware of the change. Displays the Application File Datefor the Intake case action.

·         Verification Received On:Enter the date the client provided verification.

Medicare Details

·         Type:Select Part A or Part B.

·         Claim #:Select the claim number if populated with the SSCNs entered on the Individual SSCNspage for the selected individual. Otherwise this field is read-only if there are no SSCN numbers.

Details

·         Medicare Begin Date:Enter the Medicare begin date if enabled.

·         Medicare End Date:Enter the Medicare end date.

·         Verification:Select the verification.

·         Buy-in Begin Date:Displays the buy-in begin date.

·         Buy-in End Date:Displays the buy-in end date.

Enrollment

·         Entitled to Enroll:Select Yes or No.

·         Enrolled Date:Enter the date of enrollment if the above field is Yes.

DCMEC - Medicare Claim – Summary

·         Name:Displays the full name, age, and gender of the individual.

·         Type:Displays Part A or Part B as the Medicare type.

·         Claim #:Displays the claim number.

·         Medicare Begin Date:Displays the Medicarebegin date.

·         Medicare End Date:Displays the Medicare end date.

Person Information

·         Person ID:Displays the unique ID number assigned to an individual. This field populates after the Name field is selected.

Minor Parent Dates

·         Circumstances Start/Change Date: (CS/CD) Enter the start or change date for a new circumstance. Displays the first of the month, three months prior to the Application File Date for Intake case action.

·         End Date: Enter the end date for the situation.

·         Reported On: Enter the date the client reported a change. Displays the Application File Date for Intake case action.

·         Date Client Became Aware:Enter the date the client became aware of the change. Displays the Application File Datefor the Intake case action.

Minor Parent Information

·         Minor parent lives with parent/step-parent/legal-guardian:Select Yes or No.

·         Good cause reason for not living with parent(s)/step-parent(s)/legal-guardian(s):Select the reason if the above field is No.

·         Verification:Select the verification if enabled.

DCSMP - Minor Parent – Summary

·         Name:Displays the full name, age, and gender of the individual.

·         Circumstances Start/Change Date:(CS/CD) Displays the start or change date of the circumstance.

·         End Date:Displays the end date for the historical record.

·         Lives with Parent:Displays Yes or No.

·         Good cause reason:Displays the good cause reason.

DCSMA - Special Medicaid Approvals – Details

Individual Information

·         Name:Select the individual’s name if this is a new record. Otherwise, displays the individual’s name.

·         Person ID:Displays the unique ID number assigned to an individual. This field populates after the Name field is selected.

Special Medicaid Approvals Information Dates

·         Circumstances Start/Change Date:(CS/CD) Enter the start or change date for a new circumstance. Displays the first of the month, three months prior to the Application File Date for Intake case action.

·         End Date:This field is disabled and will display the approval end date if one is entered on this page.

Special Medicaid Approvals Information

·         Approval Type:  Select the type of approval if enabled.

·         Approval Begin Date:Enter the approval begin date for all approval types except for Former Foster Care.

·         Approval End Date:Enter the approval end date for all approval types except for Former Foster Care.

·         Provider ID:Enter the provider ID if enabled.

·         Foster Care State:Select the foster care state if enabled.

·         Ward Type:Select the ward type if enabled.

·         Ward County:Select the ward county if enabled.

·         Individual was receiving Medicaid?:Select the individual if the Approval Type field is Former Foster Care.

·         How old was the individual when they left the foster care system?:Enter the age if the Approval Type field is Former Foster Care.

·         Verification:Select the verification if the Approval Type field is Former Foster Care.

·         Effective Begin Date:Displays the start or change date of the circumstance.

·         End Date:Displays the end date for the historical record.

·         Approval Type:Displays the approval type.

·         Approval Begin Date:Displays the approval begin date.

·         Approval End Date:Displays the approval end date.

DCHQD - Medically Frail - Details

Person Information

·         Name:  Displays the full name, age and gender of the individual.

·         Person ID:  Displays the unique ID number assigned to an individual.

Medically Frail Dates

·         Circumstances Start / Change Date:  (CS/CD) Displays the circumstances start/change date. This will be updated to the current date when the Medically Frail Determination Status changes.

·         End Date:  Displays an end date when an updated Medically Frail Determination Status occurs.

Medically Frail Determination Status

·         Medically Frail Determination Status:  Displays the medically frail determination status from the MCE Interface, SSA Bendex Interface or SSA SDX Interface.

·         Status Date:  Displays the status date from the MCE interface, SSA Bendex interface or SSA SDX interface.

DCHQS - Medically Frail - Summary

·         Name:  Displays the full name, age and gender of the individual.

·         Person ID:  Displays the unique ID number assigned to an individual.

·         Circumstances Start/Change Date:  (CS/CD) Displays the start or change date of the circumstance.

·         Medically Frail Determination Status:  Displays the medically frail determination status.

·         Status Date:  Displays the status date.
